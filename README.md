# elfobserver

(Based on the summarization with GEMINI CODE ASSIST)

This codebase is a suite of tools for Extremely Low Frequency (ELF) signal observation, processing, and analysis. It is designed to capture raw signal data, perform noise reduction, and analyze the signal for specific patterns using a Quadratic Phase Detector (QPD), often utilizing CUDA for GPU acceleration.

Here is a summary of the key components and files:

# 1. Real-time Observation & Orchestration (elfobserver.cpp)

This is the main application for real-time monitoring.

* Functionality: It listens for UDP packets containing raw signal data, processes them through a pipeline, and interacts with a backend analysis server.
* Pipeline:
  * Ingestion: Receives raw packets and applies a Half-Band FIR filter to downsample/condition the signal.
  * Noise Reduction: Uses a threaded pipeline to remove impulse noise, white noise, and line noise (mains hum) using NLMS (Normalized Least Mean Squares) filters.
  * Analysis Offload: Sends buffered signal data to a remote QPD server (TCP) for heavy computation.
  * Reporting: Receives analysis reports back. If a signal "rank" (signal-to-noise ratio metric) exceeds a watermark, it saves the report and the corresponding raw audio data to disk.

# 2. QPD Analysis Server (qpd.cu)

This acts as the backend computation engine for elfobserver.

*  Functionality: A TCP server that accepts signal data chunks from the observer.
* Algorithm: Implements a Quadratic Phase Detector (QPD). It convolves the input signal with a set of reference chirp functions (cosine and sine waves with varying frequency rates/alphas). These chirp functions are the part of ["whistler" waves](https://en.wikipedia.org/wiki/Whistler_\(radio\)) of which frequency are inversely propotional to the square of time.
* Acceleration: It uses CUDA kernels (conv_partial) to parallelize the heavy convolution operations on an NVIDIA GPU.
* Output: Returns a statistical report (mean value, max value, and peak coordinates) back to the client.

# 3. Offline Analysis Tools

* elfqpd.cu: A standalone tool for offline analysis. It reads a WAV file, performs the same QPD analysis as the server, and saves the detailed output matrices to an HDF5 file (/qpddataset) for visualization or further research. qpdshow.pl python script can draw the heatmap for that HDF5 file. 
* elfclean.cpp: A utility to clean up existing WAV recordings. It applies the same noise reduction chain (Impulse Detector -> Impulse NLMS -> White Noise NLMS -> Line Noise NLMS) found in the observer and outputs a "clean" WAV file.
* repoprint.cpp: A simple utility to read the binary .repo files generated by elfobserver and print them in a human-readable text format.

# Key Algorithms & Libraries

* Signal Processing: NLMS adaptive filters, Impulse Noise Detection, Half-Band FIR filtering.
* QPD: A specific detection algorithm likely looking for "chirp" signals (signals where frequency changes with time), parameterized by alpha (rate of change) and toff (time offset).
* Libraries:
  * CUDA: For GPU acceleration of convolutions.
  * Boost: Specifically boost::circular_buffer for managing signal history in filters.
  * HDF5: For storing large analysis datasets.
  * POSIX Sockets: For UDP/TCP communication between the observer and the QPD server.

# Build

```
elfobserver:
g++ --std=c++23 -Wall -O3 -o elfobserver elfobserver.cpp

qpd:
nvcc qpd.cu -o qpd -O3

elfclean:
g++ --std=c++23 -Wall -O3 -o elfclean elfclean.cpp

elfqpd:
nvcc -I/usr/include/hdf5/serial -O3 -o elfqpd elfqpd.cu -L/usr/lib/x86_64-linux-gnu/hdf5/serial -lhdf5_cpp -lhdf5

repoprint:
g++ --std=c++23 -Wall -O3 -o repoprint repoprint.cpp
```
